# DIGITALIQ — UI / UX EXECUTION BRIEF

Use the provided website content and sitemap to redesign and build the website as a premium, modern, production-grade experience.

The final result must not look like a template, a typical agency theme, or an AI-generated layout.  
It should feel expensive, custom, highly intentional, and visually refined — the kind of website that makes people say: **this looks like a $100k website**.

---

## DESIGN SYSTEM LOCK

### Accent Color
- Primary accent color: `#FFE100`
- Use this as the signature highlight color across the experience
- It should feel powerful and intentional, never overused

### Typography
- Headline font: `Satoshi`
- Body font: `Inter`

Important:
- `Satoshi` is not a Google Font default; implement it properly from a valid source
- `Inter` should be used only for body text, supporting copy, and functional readability
- `Satoshi` should carry the visual identity of the site

### Core Visual Tone
The website should feel like a fusion of:
- futuristic precision
- luxury-level restraint
- editorial sophistication
- high-end product design execution

---

## OVERALL EXPERIENCE

The site should feel:
- ultra-modern
- premium
- bold but controlled
- clean, sharp, and highly intentional
- visually impressive without becoming messy
- advanced in execution, not flashy in a cheap way

Avoid:
- generic SaaS design
- obvious AI aesthetics
- basic startup gradients
- overused agency layouts
- template-style section stacking
- anything that feels mass-produced

---

## CURSOR / MOUSE INTERACTION

The mouse interaction should use the accent color `#FFE100`.

If a custom cursor is implemented:
- it must feel premium and smooth
- it must not feel gimmicky or distracting
- it must not reduce usability
- it should respond elegantly to hover states and clickable elements
- subtle scale, glow, blend, or trailing behavior is acceptable only if it feels refined

If a full custom cursor feels too heavy, use accent-color hover response and micro-interactions instead.

The result must feel expensive, not playful.

---

## OPENING ANIMATION

When the site first opens:

- the **DigitalIQ logo should appear centered on the screen**
- this moment should feel cinematic, premium, and smooth
- after that, the site should reveal itself elegantly

Rules:
- keep the intro short and polished
- make it feel like a luxury digital reveal
- do not delay the user too long
- transition smoothly into the homepage
- avoid cheesy or dramatic effects

---

## COLOR & SURFACE SYSTEM

Build a refined dark-mode-first interface.

Suggested direction:
- deep dark background tones
- layered dark surfaces
- subtle tonal contrast
- accent color `#FFE100` used with discipline

Use CSS variables / design tokens for the color system.

The site should not rely on loud gradients.
Backgrounds should feel rich through:
- subtle texture
- restrained atmosphere
- depth
- shadows
- layered surfaces
- controlled glow where appropriate

---

## TYPOGRAPHY RULES

- `Satoshi` should be used for hero headlines, section titles, navigation emphasis, and important UI moments
- `Inter` should be used for body text, supporting copy, and readable content blocks
- Headlines must feel commanding and premium
- Body text must feel clean, crisp, and highly readable
- Use a clear type hierarchy and keep it consistent throughout

Avoid:
- typography that feels generic
- weak headline presence
- oversized body text that feels template-like

---

## LAYOUT & COMPOSITION

The layout must feel custom-designed.

Use:
- strong structure
- intentional rhythm
- sophisticated spacing
- controlled asymmetry where appropriate
- occasional overlap or broken-grid moments only when they improve visual interest
- sections that feel distinct but still part of one system

Avoid:
- repetitive template blocks
- predictable section stacking
- generic card-grid sameness
- chaotic layouts pretending to be creative

The site should feel engineered and art-directed at the same time.

---

## COMPONENT DESIGN

Every component must feel custom-designed, modern, and premium.

This includes:
- navbar
- buttons
- cards
- links
- dropdowns
- forms
- service blocks
- section transitions
- footer
- CTA zones

The component system should feel current, high-end, and visually refined — not generic, flat, or template-like.

Use:
- premium surface layering
- refined border treatments
- subtle transparency where appropriate
- sharp hover and active states
- elegant shadow hierarchy
- visually rich CTA treatments
- modern card construction with depth and intention
- polished state transitions
- highly considered spacing and alignment

Important:
- components should look like they were designed by a senior product designer, not assembled from a UI kit
- avoid generic cards, default buttons, and obvious template patterns
- interactive states should feel advanced and satisfying
- the overall component language should feel trend-aware, but timeless enough to remain premium

Everything should follow one unified design system:
- consistent radius
- consistent spacing scale
- consistent shadow logic
- consistent hover behavior
- polished transitions
- clear active states using the accent color

No generic components.
No default-looking UI kit styling.

---

## MOTION & INTERACTION

Motion must feel premium and intentional.

Include:
- elegant entrance animations
- staggered reveals where appropriate
- refined hover states
- smooth transitions between sections
- subtle scroll-triggered motion if it improves the experience

Motion should feel:
- smooth
- modern
- expensive
- controlled

Avoid:
- excessive motion
- gimmicky parallax overload
- flashy effects that hurt readability
- animation for animation’s sake

---

## RESPONSIVENESS

Mobile responsiveness must feel just as premium as desktop.

Do not reduce the mobile version into a boring simplified layout.
Keep the same visual quality, hierarchy, and polish across breakpoints.

The experience must feel:
- premium on desktop
- premium on tablet
- premium on mobile

---

## QUALITY BAR

The final result should feel like:
- a world-class custom agency website
- a premium digital product experience
- something designed by a senior product designer and front-end creative developer

It should NOT feel like:
- AI-generated sections
- a basic Framer-style template
- a startup landing page
- a generic agency portfolio

The goal is a website that feels unforgettable, expensive, modern, and unmistakably custom.
