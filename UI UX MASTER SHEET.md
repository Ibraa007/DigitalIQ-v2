# UI UX MASTER SHEET.md — DigitalIQ Website System

## 0. PURPOSE
This file defines the visual, interaction, and experience rules for the DigitalIQ website.

It is a source of truth for how the website should look, feel, and behave during execution.

All design and frontend decisions must follow this file.

This website must feel:
- premium
- modern
- highly intentional
- custom-built
- visually refined
- commercially serious

This website must not feel:
- templated
- generic
- AI-generated
- startup-SaaS-like
- overused agency-style
- mass-produced

---

## 1. BRAND EXPERIENCE GOAL

The DigitalIQ website should feel like a high-end custom digital product built for a serious brand.

The result should make people feel:
- this is premium
- this is modern
- this is intentional
- this is not an AI-made site
- this looks like a very expensive custom build

The visual tone should feel like a fusion of:
- futuristic precision
- luxury-level restraint
- editorial sophistication
- high-end product design execution

---

## 2. DESIGN SYSTEM LOCK

### Accent Color
- Primary Accent: `#FFE100`

Use the accent color as a signature highlight only.

Approved use cases:
- primary CTA emphasis
- hover states
- active states
- cursor interaction
- selected visual highlights
- premium detail accents

The accent must feel intentional, never excessive.

### Typography
- Headline Font: `Satoshi`
- Body Font: `Inter`

Rules:
- Satoshi carries the brand identity
- Inter is used for body text and readable supporting content
- Do not treat both fonts equally
- Headlines should feel commanding
- Body copy should feel clean, crisp, and highly readable

### Dark-Mode-First
The site should be designed dark-mode-first.

The overall visual direction should use:
- deep dark backgrounds
- layered dark surfaces
- tonal contrast
- depth
- subtle glow
- controlled highlights

Do not rely on loud gradients or bright generic color fields.

---

## 3. VISUAL STYLE RULES

The website should feel:
- sharp
- engineered
- premium
- controlled
- visually rich without clutter
- modern without trend-chasing

Use:
- clean structure
- rich but restrained depth
- strong contrast
- refined layering
- premium surface treatment
- controlled asymmetry when valuable
- elegant visual hierarchy

Avoid:
- flat generic layouts
- obvious template section stacking
- excessive color noise
- playful aesthetics
- cheap futuristic clichés
- random decorative effects

---

## 4. LOGO INTRO RULE

When the site first opens:

- the DigitalIQ logo must appear centered on the screen
- this must feel cinematic, smooth, and premium
- the reveal must be short and polished
- the transition into the homepage must feel elegant

Do not:
- make the intro too long
- make it dramatic or cheesy
- make it feel like a game intro
- delay access to the site unnecessarily

The effect should feel like a luxury digital reveal.

---

## 5. CURSOR / POINTER INTERACTION RULE

The pointer interaction must reflect the accent color `#FFE100`.

If using a custom cursor:
- it must feel premium
- it must remain usable
- it must not distract from readability
- it must not feel playful or gimmicky
- it should respond smoothly to interactive elements

Acceptable behaviors:
- subtle scale response
- soft glow
- hover adaptation
- refined pointer transformation on links or buttons

If a full custom cursor harms usability, use refined accent-colored hover and interaction feedback instead.

The interaction must feel expensive, not novelty-driven.

---

## 6. LAYOUT SYSTEM

Layouts must feel custom-designed, not assembled.

Use:
- strong structure
- intentional spacing rhythm
- controlled negative space
- section pacing that feels editorial and premium
- occasional broken-grid or overlap moments only when they improve hierarchy or interest
- clear separation between sections while maintaining one unified system

Avoid:
- repetitive block stacking
- identical section formulas
- generic grid repetition
- visual monotony
- chaos pretending to be creativity

The website should feel art-directed and engineered at the same time.

---

## 7. SECTION RHYTHM

Every page should have rhythm.

That means:
- strong opening
- visual rise and release
- variation in density
- some sections feeling more immersive
- some sections feeling more operational
- some sections feeling more focused and conversion-driven

Do not make every section feel visually equal.

The most important sections should carry more visual weight.

---

## 8. COMPONENT SYSTEM

Every component must feel custom-designed, modern, and premium.

This includes:
- navbar
- hero actions
- buttons
- cards
- links
- dropdowns
- forms
- service blocks
- CTA sections
- footer
- section dividers
- content modules

Components must feel:
- current
- high-end
- detailed
- polished
- custom-made

Use:
- premium surface layering
- refined border treatments
- elegant shadow hierarchy
- considered spacing and alignment
- sharp hover and active states
- subtle transparency when appropriate
- modern construction with visual depth
- polished transitions

Avoid:
- default UI kit styling
- generic cards
- flat boring buttons
- unconsidered states
- basic template components

No component should look default.

---

## 9. COMPONENT STATE RULES

Every interactive element must have:
- default state
- hover state
- focus-visible state
- active state

These states must feel intentional and premium.

Do not use:
- `transition-all`
- random motion
- default browser-feeling states
- weak or invisible focus logic

Hover and active states should feel:
- smooth
- sharp
- satisfying
- controlled

Accent color should appear selectively in important interactive states.

---

## 10. TYPOGRAPHY RULES

### Headlines
Headlines must feel:
- strong
- sharp
- premium
- deliberate
- high-impact

Use Satoshi for:
- hero headings
- section headings
- key labels
- emphasized UI headings
- navigation emphasis where needed

### Body Copy
Body copy must feel:
- clean
- modern
- readable
- premium
- calm

Use Inter for:
- paragraphs
- supporting copy
- descriptive content
- UI helper text
- form-related text

### Hierarchy
Type hierarchy must be clear and consistent.

Do not:
- flatten the difference between heading and body text
- use oversized body text that feels template-like
- allow headings to feel weak or generic

---

## 11. COLOR & SURFACE RULES

Dark-mode-first visual system must use:
- deep background surfaces
- secondary layered surfaces
- floating elevated surfaces
- restrained accent application
- subtle tonal contrast

Backgrounds should feel rich through:
- depth
- shadow
- light layering
- controlled atmosphere
- subtle texture if needed
- occasional restrained glow

Avoid:
- plain empty dark backgrounds with no richness
- loud rainbow or startup gradients
- visually noisy backgrounds
- overly glossy fake-futuristic effects

---

## 12. SHADOW, DEPTH, AND ELEVATION

The interface must have a real depth system.

Define a clear sense of:
- base layer
- elevated layer
- floating layer

Use shadows and surface treatment to separate these states.

Shadows should feel:
- layered
- soft but intentional
- premium
- integrated with the design

Avoid:
- flat `shadow-md` style treatment
- one identical shadow everywhere
- fake depth with harsh low-quality effects

---

## 13. MOTION RULES

Motion must feel:
- smooth
- premium
- controlled
- modern
- expensive

Include:
- elegant entrance animations
- subtle staggered reveals where useful
- refined hover interactions
- smooth section transitions
- restrained scroll-triggered motion where it adds value

Avoid:
- excessive motion
- parallax overload
- bouncy or playful movement
- flashy gimmicks
- animation for animation’s sake

Only animate in ways that improve the experience.

Motion should support hierarchy, not distract from it.

---

## 14. RESPONSIVENESS RULES

The site must remain premium across:
- desktop
- tablet
- mobile

Mobile must not feel like a downgraded version.

Keep:
- hierarchy
- visual quality
- polish
- spacing discipline
- component refinement

Do not collapse the experience into a generic mobile layout.

---

## 15. PAGE-SPECIFIC EXPERIENCE RULES

## HOME
The homepage must feel:
- premium
- brand-led
- structured
- high-conviction

Rules:
- the hero must be the strongest visual moment on the site
- the page must feel like a serious brand, not a typical agency homepage
- the method section should feel intelligent and easy to follow
- the services overview should feel like one connected ecosystem, not a simple list

## ABOUT
The About page must feel:
- credible
- institutional
- premium
- serious

Rules:
- the page should communicate origin, company depth, and trust
- it should feel like brand credibility, not a casual profile page

## METHOD
The Method page must feel:
- structured
- operational
- intelligent
- easy to understand

Rules:
- the four stages should be visually clear
- sequence should feel important
- the page should feel like a real operating model, not a decorative process page

## SERVICES
The Services page must feel:
- systemic
- structured
- premium
- connected

Rules:
- the Intelligence Layer must feel foundational and elevated
- the five engines must feel distinct but interconnected
- the page must feel like an ecosystem, not a menu
- flexibility and tailored application should still feel clear

## START NOW
The Start Now page must feel:
- premium
- selective
- structured
- trustworthy

Rules:
- it must not feel like a normal contact page
- the 3-step process should be visually clear
- it should feel like the beginning of a serious engagement

## ENGINE PAGES
Engine pages must feel:
- focused
- commercially clear
- premium
- consistent with the full system

Rules:
- each engine should feel distinct in emphasis
- each engine must still clearly belong to the same website system
- do not let engine pages become repetitive or generic

---

## 16. CONTENT PRESENTATION RULES

Content must not be presented as static walls of text.

Translate content into:
- premium layout modules
- structured text blocks
- hierarchy-led groupings
- visual content zones
- intelligent spacing systems

Do not invent new messaging.

Do not rewrite the content direction unless explicitly asked.

The role of design is to elevate and structure the approved content.

---

## 17. QUALITY CONTROL RULES

After major changes:
- review the screen carefully
- assess spacing, hierarchy, alignment, and visual quality
- check that the site still feels premium
- refine weak sections before considering the work done

Always ask:
- does this feel custom?
- does this feel expensive?
- does this feel modern?
- does this feel intentional?
- does this feel like DigitalIQ?

If the answer is no, continue refining.

---

## 18. HARD NEVER RULES

Never:
- use generic SaaS aesthetics
- use obvious template layouts
- use default-looking components
- use weak typography hierarchy
- use excessive accent color
- use cheap futuristic visuals
- use playful animation
- use flat unlayered surfaces
- make the site feel like an AI-generated mockup
- make Start Now feel like a normal contact page
- make Services feel like a standard service grid
- make Home feel generic

---

## 19. FINAL STANDARD

The final website should feel like:
- a premium digital brand experience
- a custom product-level website
- a serious modern company with strong taste
- something designed by senior designers and developers

The final website should not feel like:
- a quick AI build
- a theme
- a common agency landing page
- a generic startup site

The quality bar is simple:

If someone sees the website, the reaction should be:
**this looks expensive, custom, and exceptionally well designed.**
